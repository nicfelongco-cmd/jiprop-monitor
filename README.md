# JIPROP — J Tower Title Conversion Monitor

Cross-platform app: iOS (App Store) · Android (Google Play) · Web browser.

## Project Structure

```
real-estate-monitor/
  www/                  ← All web assets (Capacitor's webDir)
    index.html          ← The app (modified from root index.html)
    assets/
      js/
        chart.umd.min.js  ← Bundled Chart.js (no CDN in native builds)
  ios/                  ← Generated by `npx cap add ios`
  android/              ← Generated by `npx cap add android`
  capacitor.config.ts   ← Capacitor configuration
  package.json
```

## Responsive Breakpoints

| Screen            | Width       | Layout                                   |
|-------------------|-------------|------------------------------------------|
| Phone             | < 768 px    | Bottom tab bar, stacked columns, 2-col KPIs |
| Tablet            | 768–1199 px | Top nav, 2-col entry panel, stacked dist row |
| Laptop / Desktop  | 1200–1440 px| Full layout, top nav                    |
| Large monitor     | > 1440 px   | **Locked at 1440 px**, centered          |

```

```

## Build Native Apps

### Prerequisites
- [Node.js 18+](https://nodejs.org)
- **iOS:** macOS + Xcode 15+ + Apple Developer account ($99/yr)
- **Android:** Android Studio + Google Play Developer account ($25 one-time)

### Initialize native projects (first time only)

```bash
npx cap add ios
npx cap add android
```

### Sync web assets to native projects

Run this every time you update `www/index.html`:

```bash
npm run cap:sync
```

### Open in Xcode (iOS)

```bash
npm run cap:ios
```

Then in Xcode:
1. Select your team under **Signing & Capabilities**
2. Set Bundle ID to `com.jiprop.titlemonitor` (or your own)
3. **Product → Archive** to submit to App Store

### Open in Android Studio

```bash
npm run cap:android
```

Then in Android Studio:
1. **Build → Generate Signed Bundle/APK** to build for Play Store
2. App ID is `com.jiprop.titlemonitor`

## App Store / Play Store Assets Needed

- App icon: 1024×1024 px PNG (no rounded corners — stores apply them)
- Splash screen: see `capacitor.config.ts` for `SplashScreen` settings
- Screenshots: at least 2 per device size (Capacitor's live preview helps)

## Notes

- **Fonts**: DM Sans + DM Mono load from Google Fonts CDN. Devices need internet
  on first launch to cache fonts. For fully-offline builds, download the WOFF2
  files and add `@font-face` declarations pointing to `assets/fonts/`.
- **Data storage**: `localStorage` + `IndexedDB` — persists within the app.
  Data does not sync between devices (no backend).
- **Print/Export PDF**: works in the browser. On iOS/Android the system share
  sheet appears instead of a print dialog (standard WebView behavior).
